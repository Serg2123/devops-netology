10. по идее так, touch sample-{1..100000}.txt, но у меня даже 90000 не получилось :), методом подбора остановился на 87500 файлах, все упирается в ограничения аргументов уровня ядра echo ARG_MAX, но гугл знает все и есть утилита (xargs) которая понимает данную проблему и сама постранично формирует вызов с нужными параметрами: echo sample{1..300000}.txt | xargs touch
Так что да, одной командой можно создать 300.000 файлов, и даже больше. И rm, кстати, из-за аналогичной проблемы не может удалить разом 300.000 файлов, а вот оболочка mc легко.

11. выражение-конструкция, которая на выходе дает true или false по результату проверки наличия папки /tmp

12. в /etc/environment в самое начало переменной PATH добавим (без кавычек) "/tmp/new_path_directory:" 

далее создадим в темпе новую символическую папку из исходной в которой размещен bash:

sudo ln -s /usr/bin /tmp

переместимся в tmp и переименуем полученную симолическую папку в нужное название:

sudo mv bin new_path_directory

перезайдем в сессию
и после этого bash начинает искаться первым в новой папке:

vagrant@vagrant:~$ type -a bash

bash is /tmp/new_path_directory/bash

bash is /usr/bin/bash

bash is /bin/bash
